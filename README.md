# 텀블벅 후원 트래커 (Google Sheets 연동)

텀블벅에서 후원한 프로젝트를 **구글 시트를 저장소로** 쓰면서 웹앱에서 관리하는 도구입니다.
한 번 배포하면 PC·모바일·태블릿 어디서나 같은 데이터를 볼 수 있어요.

## 기능

- 후원 목록 (제목 / 금액 / 상태 / 창작자 / 카테고리)
- 결제 예정일 타임라인 (D-day 표시)
- 다운로드·수령 완료 체크박스
- 카테고리·월별·상태별·수령 통계
- 텀블벅 후원 현황 페이지 **복사 → 붙여넣기** 자동 파싱
- 모든 데이터는 **내 구글 시트에만** 저장됨

## 설치 방법 (5분)

### 1. 새 구글 시트 만들기

<https://sheets.new> 접속 → 빈 시트 생성. 파일 이름은 자유롭게 (예: `텀블벅 후원 기록`).

### 2. Apps Script 열기

시트 상단 메뉴: **확장 프로그램 → Apps Script**

### 3. 코드 붙여넣기

Apps Script 편집기에서:

**a. `Code.gs` 붙여넣기**
- 기본으로 열려있는 `Code.gs` 파일의 내용을 모두 지우기
- 이 저장소의 [`Code.gs`](./Code.gs) 전체 내용을 복사해서 붙여넣기
- 저장 (💾 아이콘 또는 `Ctrl/⌘+S`)

**b. `Index.html` 추가**
- 왼쪽 파일 목록에서 `+` → **HTML** 선택
- 이름을 `Index` 로 입력 (확장자 `.html` 자동 추가)
- 이 저장소의 [`Index.html`](./Index.html) 전체 내용을 복사해서 붙여넣기
- 저장

**c. (선택) `appsscript.json` 수정**
- 편집기 좌측 상단 ⚙ **프로젝트 설정** → "'appsscript.json' 매니페스트 파일을 편집기에 표시" 체크
- 다시 편집기로 돌아가 `appsscript.json`의 내용을 이 저장소의 파일로 교체

### 4. 시트 초기화

Apps Script 편집기에서:
- 상단 함수 선택 드롭다운에서 `setupSheet` 선택
- ▶ **실행** 클릭
- 첫 실행 시 구글 권한 요청이 뜹니다 → **검토 → 본인 계정 선택 → "고급" → "(안전하지 않음으로) 이동" → 허용**
  - ⚠️ "안전하지 않음" 경고는 본인이 만든 스크립트라 뜨는 정상 현상입니다.
- 시트로 돌아가 보면 `projects` 시트가 생성되고 헤더가 설정됨

### 5. 웹앱으로 배포

Apps Script 편집기 우측 상단:
- **배포 → 새 배포**
- 톱니바퀴 ⚙ → **웹 앱** 선택
- 설명: 자유롭게 (예: "v1")
- **다음 사용자로 실행**: `나`
- **액세스 권한이 있는 사용자**: `나만` (본인 계정만 사용)
- **배포** 클릭
- 나오는 **웹 앱 URL**을 복사해서 북마크 해두기

완료! 그 URL이 여러분의 개인 텀블벅 트래커입니다.

## 사용법

### 모바일에서 빠르게 열기

방금 얻은 웹앱 URL을 모바일 브라우저로 열고 **홈 화면에 추가**하면 앱처럼 바로 쓸 수 있어요.

### 붙여넣기로 일괄 등록

1. 텀블벅 로그인 → 프로필 → **후원 현황(후원한 프로젝트)**
2. 후원 목록 영역을 드래그해서 복사
3. 트래커에서 **붙여넣기로 추가** → 붙여넣기 → **분석하기** → **시트에 저장**

제목·금액·결제일·상태가 자동으로 추출됩니다. 부족한 정보는 편집에서 채우세요.

### 시트에서 직접 편집

구글 시트를 열어서 셀 값을 직접 고쳐도 됩니다. 웹앱 새로고침(↻) 버튼을 누르면 반영돼요.

### 여러 기기·동시 편집

같은 구글 계정으로 로그인하면 어디서든 같은 데이터를 봅니다.
시트를 다른 사람과 공유하면 공동 편집도 가능 (단 웹앱 접근 권한은 별도 설정 필요).

## 문제 해결

**"페이지를 찾을 수 없음" / 권한 오류**
- 처음 배포 시 권한 허용이 필요합니다. 4번의 `setupSheet` 실행부터 다시 해보세요.

**붙여넣기 후 분석 결과가 이상해요**
- 텀블벅 페이지 레이아웃은 바뀔 수 있어요. 금액(`N,000원`)이 텍스트에 포함되어 있는지 확인.
- 한 번에 잘 안 되면 프로젝트별로 나눠서 여러 번 붙여넣어도 됩니다 (중복 자동 제거).

**코드를 수정했는데 반영이 안 돼요**
- 웹앱은 "새 배포"를 만들어야 코드 변경이 반영됩니다.
- **배포 → 배포 관리 → 연필 ✏ → 버전: 새 버전 → 배포** 를 하면 같은 URL로 최신 버전이 반영돼요.

## 개인정보

- 데이터는 전부 **본인 구글 시트**에만 저장됩니다.
- 텀블벅 로그인 정보는 일체 사용하지 않아요 (수동 붙여넣기 방식).
- 웹앱 기본 설정은 `나만 접근 가능`이라서 URL이 노출돼도 타인은 사용 불가.
